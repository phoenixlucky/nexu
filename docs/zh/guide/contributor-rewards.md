# 贡献奖励与支持

如果你为 **nexu** 做出贡献，我们希望这件事不只是“PR merge 了”，还包括：

- 你的每一次认真投入，都能被看见
- 你的贡献，不会只停留在 merge 记录里
- 你的成长，会被持续记录和积累
- 你第一次参与时，也能感受到社区的欢迎

## 我们会提供什么帮助

对于 `good-first-issue` 和首次贡献者，我们会尽量提供这些支持：

- **更清晰的题目范围**：尽量把题目拆成小而清晰、可验收的任务
- **更明确的 review 反馈**：如果需要修改，我们会尽量说明原因和方向
- **首次贡献引导**：第一次参与时，尽量给你必要的说明和建议
- **公开致谢与积分记录**：合入后会进入贡献者致谢与积分流程

如果你还没开始，可以先看：

- [第一次提 PR 指南](/zh/guide/first-pr)
- [参与贡献](/zh/guide/contributing)
- [Good First Issue 列表](https://github.com/nexu-io/nexu/labels/good-first-issue)

## 你的贡献会展示在哪里

如果你的贡献被合入，后续会展示在这些位置：

1. **GitHub README**：展示贡献者，并同步呈现 **日榜、月榜和历史累计榜单**
2. **Discord [Start your first Nexu PR](https://discord.gg/vMrySTJW8u)**：Bot 自动发布排行榜动态

## 当前积分规则

以下为当前口径；若后续团队公告调整，以最新公示为准：

| 贡献类型 | 积分 | 说明 |
| --- | --- | --- |
| PR 合入（starter 难度） | 10 | 修复 typo、配置调整 |
| PR 合入（easy 难度） | 25 | Bug 修复、小功能 |
| PR 合入（medium 难度） | 50 | 完整功能实现 |
| PR 合入（advanced 难度） | 100 | 架构级改动 |
| Skill 发布到 ClawHub | 80 | 独立可运行的 Skill |
| Bug Report（被确认） | 10 | 提供复现步骤的 Bug 报告 |
| 文档贡献 | 15-30 | 按篇幅和深度 |
| 翻译贡献 | 20 / 语言 | 每个语言包 |
| Code Review（被采纳） | 15 | 有实质性建议的 Review |
| Discussion 优质回答 | 10 | 被标记为 Answer 的回复 |
| 社区推荐新贡献者 | 20 | 推荐人首次 PR 合入后给推荐者 |

## 这些积分意味着什么

积分不是为了“刷分”，而是为了把每一种有价值的贡献认真记录下来。

- 小修小补也有价值
- 文档、翻译、Review 不再被忽略
- 首次贡献者能更明确看到正反馈
- 长期贡献者能在社区里积累可见身份
- 积分可以兑换成社区徽章
- 积分也会沉淀成你的开源贡献履历

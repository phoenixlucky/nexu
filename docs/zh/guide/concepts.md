# 核心概念

## Agent

Agent 是 nexu 的核心运行单元——一个持续在线的 AI 助手，能够同时接入多个聊天平台，理解上下文并执行任务。

每个工作区对应一个 Agent 实例。

## 渠道（Channels）

渠道是 Agent 与用户交互的入口。nexu 支持飞书、Slack、Discord 三个平台，填入对应凭证即可完成接入。

同一个 Agent 可以同时连接多个渠道，消息在各平台间独立处理。

## 模型（Models）

模型决定了 Agent 的推理能力。nexu 提供两种接入方式：

- **nexu Official** — 开箱即用，无需配置
- **BYOK** — 接入你自己的 Anthropic、OpenAI、Google AI 或其他兼容服务商

支持随时切换，不影响已有对话。

## 技能（Skills）

技能是 Agent 的能力扩展机制。每个技能是一个独立模块，赋予 Agent 特定的执行能力——数据查询、文档生成、第三方服务调用等。

支持从技能目录一键安装，也支持本地开发。

## 部署（Deployments）

nexu 采用桌面端本地部署模式。启动客户端、连接渠道，Agent 即刻就绪。

无需额外的服务器或容器编排。

## 工作区（Workspace）

工作区是 nexu 中所有配置的归属单元，关联了 Agent、渠道、模型和技能。一个工作区即一套完整的运行环境。

# 기여자 보상 및 지원

**nexu**에 기여할 때, "PR이 머지되었다"는 것 이상의 의미를 담고 있습니다:

- 여러분의 진지한 노력이 보이는 형태로 남습니다
- 기여가 머지 기록에만 머물지 않습니다
- 성장이 지속적으로 기록되고 축적됩니다
- 첫 참여에서도 커뮤니티의 환영을 느낄 수 있습니다

## 우리가 제공하는 지원

`good-first-issue`와 첫 기여자에게 다음과 같은 지원을 제공합니다:

- **더 명확한 작업 범위**: 작고 명확하며 검증 가능한 작업으로 분할
- **더 구체적인 리뷰 피드백**: 변경이 필요한 경우 이유와 방향을 설명
- **첫 기여 안내**: 첫 참여 시 필요한 설명과 조언 제공
- **공개 감사 및 포인트 기록**: 머지 후 기여자 감사 및 포인트 프로세스에 진입

아직 시작하지 않았다면:

- [Good First Issue 기여자 가이드](/ko/guide/first-pr)
- [기여하기](/ko/guide/contributing)
- [Good First Issue 목록](https://github.com/nexu-io/nexu/labels/good-first-issue)

## 기여가 표시되는 곳

기여가 머지되면 다음 위치에 표시됩니다:

1. **GitHub README**: 기여자 표시, **일간·월간·누적 리더보드** 동기화 표시
2. **Discord [Start your first Nexu PR](https://discord.gg/vMrySTJW8u)**: Bot이 리더보드를 자동 게시

## 현재 포인트 규칙

다음은 현재 기준입니다. 팀이 조정을 발표하면 최신 버전이 우선합니다:

| 기여 유형 | 포인트 | 설명 |
| --- | --- | --- |
| PR 머지 (starter 난이도) | 10 | typo 수정, 설정 조정 |
| PR 머지 (easy 난이도) | 25 | 버그 수정, 작은 기능 |
| PR 머지 (medium 난이도) | 50 | 완전한 기능 구현 |
| PR 머지 (advanced 난이도) | 100 | 아키텍처 수준 변경 |
| Skill을 ClawHub에 게시 | 80 | 독립 실행 가능한 Skill |
| Bug Report (확인됨) | 10 | 재현 단계가 포함된 버그 보고 |
| 문서 | 15-30 | 분량과 깊이에 따라 |
| 번역 | 20 / 언어 | 각 언어 팩 |
| Code Review (채택됨) | 15 | 실질적 제안이 있는 Review |
| Discussion 양질의 답변 | 10 | Answer로 표시된 답변 |
| 신규 기여자 추천 | 20 | 추천된 분의 첫 PR 머지 후 추천자에게 |

## 포인트의 의미

포인트는 "점수 올리기"가 아니라 가치 있는 기여를 하나하나 진지하게 기록하기 위한 것입니다.

- 작은 수정도 가치가 있습니다
- 문서, 번역, Review가 간과되지 않습니다
- 첫 기여자가 더 명확한 긍정적 피드백을 받습니다
- 장기 기여자가 커뮤니티에서 가시적인 아이덴티티를 구축합니다
- 포인트는 커뮤니티 배지로 교환 가능
- 포인트는 오픈소스 기여 이력으로 축적됩니다

## 가장 쉬운 작업부터 시작하고 싶다면

**Good First Issue** 기여자를 지속적으로 모집하고 있습니다.

이런 작업은 보통 범위가 작고 방향이 집중되어 있어 첫 기여에 적합합니다.

작업 선택하기: [Good First Issue 목록](https://github.com/nexu-io/nexu/labels/good-first-issue)

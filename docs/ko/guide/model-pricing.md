# 모델 크레딧 소비

Nexu Official 모델을 사용할 때, 각 대화는 모델 유형과 토큰 사용량에 따라 크레딧이 차감됩니다. 이 페이지에서는 현재 사용 가능한 모든 공식 모델의 크레딧 소비를 나열하여, 적절한 모델을 선택하고 크레딧 사용을 계획하는 데 도움을 줍니다.

## 모델별 크레딧 소비

크레딧은 입력 토큰과 출력 토큰에 대해 별도로 청구되며, 모델마다 요율이 다릅니다. 아래 표는 대화당 예상 비용이 낮은 순서대로 정렬되어 있습니다.

## 대화 한 번에 크레딧이 얼마나 드나요?

일반적인 단일 턴 대화를 예로 들면 — 약 2,000개의 입력 토큰(시스템 프롬프트와 컨텍스트 포함)과 2,000개의 출력 토큰:

| 모델 | 요율 | 대화당 예상 비용 | 100 크레딧당 대화 수 |
| --- | ---: | ---: | ---: |
| deepseek-v3.2 | 0.18x | ≈ 0.13 크레딧 | ≈ 769 |
| minimax-m2.7 | 0.43x | ≈ 0.30 크레딧 | ≈ 333 |
| gemini-3.1-flash-lite-preview | 0.50x | ≈ 0.35 크레딧 | ≈ 285 |
| kimi-k2.5 | 0.60x | ≈ 0.42 크레딧 | ≈ 238 |
| glm-5 | 0.86x | ≈ 0.60 크레딧 | ≈ 166 |
| **gemini-3-flash-preview (기본)** | **1.00x** | ≈ 0.70 크레딧 | ≈ 142 |
| mimo-v2-pro | 1.14x | ≈ 0.80 크레딧 | ≈ 125 |
| glm-5-turbo | 1.49x | ≈ 1.04 크레딧 | ≈ 96 |
| gpt-5.4-mini | 1.50x | ≈ 1.05 크레딧 | ≈ 95 |

> 비용이 낮은 순서대로 정렬된 예상치입니다. 실제 소비는 대화 길이, 컨텍스트 크기, 모델 출력에 따라 달라집니다. 복잡한 작업이나 다중 턴 대화는 토큰을 훨씬 더 많이 사용합니다.

## 적합한 모델 선택 방법

- **최대 절약**: deepseek-v3.2와 minimax-m2.7은 대화당 비용이 가장 낮으며, 간단한 Q&A, 일상 채팅, 크레딧이 제한된 경우에 적합합니다.
- **균형 잡힌 선택**: gemini-3.1-flash-lite-preview, kimi-k2.5, glm-5는 기능과 비용의 균형이 좋으며, 대부분의 일상 작업에 적합합니다.
- **품질 우선**: gemini-3-flash-preview, mimo-v2-pro, glm-5-turbo, gpt-5.4-mini는 더 강력한 기능을 제공하며, 코딩 지원, 심층 분석 등 요구가 높은 작업에 적합합니다.

## 더 많은 크레딧을 얻는 방법

nexu는 크레딧을 얻을 수 있는 다양한 방법을 제공합니다. 자세한 내용은 [공유로 크레딧 받기](/ko/guide/rewards)를 참조하세요.

## FAQ

**Q: 크레딧은 어떻게 차감되나요?**

각 대화는 실제 소비된 입력 토큰과 출력 토큰을 기준으로 과금됩니다. 두 항목의 합이 해당 대화의 총 크레딧 비용입니다.

**Q: 입력 토큰과 출력 토큰의 차이는 무엇인가요?**

입력 토큰에는 사용자의 메시지, 시스템 프롬프트, 대화 컨텍스트가 포함됩니다. 출력 토큰은 모델이 생성한 콘텐츠입니다. 출력 토큰은 일반적으로 콘텐츠 생성에 더 많은 연산이 필요하기 때문에 비용이 더 높습니다.

**Q: 같은 대화인데 왜 모델에 따라 비용 차이가 큰가요?**

모델마다 기능과 연산 비용이 다릅니다. 더 강력한 모델(예: gpt-5.4-mini, glm-5-turbo)은 대화당 비용이 높고, 경량 모델(예: deepseek-v3.2, minimax-m2.7)은 간단한 작업에 매우 저렴합니다.

**Q: BYOK (Bring Your Own Key) 모드에서 크레딧이 소비되나요?**

아니요. BYOK 모드에서는 자체 API 키를 사용하여 모델 프로바이더에 직접 호출합니다. 요금은 프로바이더에서 청구되며, nexu 크레딧은 소비되지 않습니다.
